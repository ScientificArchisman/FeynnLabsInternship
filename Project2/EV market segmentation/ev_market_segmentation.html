The dataset contains information about various electric cars. Here are the columns and a brief description of each:

  1. Brand: Brand of the car.
  2. Model: Model name of the car.
  3. Accel: Acceleration (time taken to go from 0 to 100 km/h).
  4. TopSpeed: Maximum speed the car can achieve.
  5. Range: Distance the car can travel on a full charge.
  6. Efficiency: Energy consumption per kilometer.
  7. FastCharge: Charging speed when using a fast charger.
  8. RapidCharge: Indicates if rapid charging is possible.
  9. PowerTrain: Type of drivetrain (e.g., All Wheel Drive, Rear Wheel Drive).
  10. PlugType: Type of plug used for charging.
  11. BodyStyle: Style of the car's body (e.g., Sedan, Hatchback).
  12. Segment: Segment classification of the car (e.g., C, D).
  13. Seats: Number of seats in the car.
  14. PriceEuro: Price of the car in Euros.

Using Unsupervised KNN Clustering

Here's our plan:

  1. Data Preparation: We'll prepare our dataset by selecting relevant features and scaling them since clustering algorithms are sensitive to the scale of data.
  2. Determine Optimal Number of Clusters: Using the Elbow Method and Silhouette Score.
  3. K-means Clustering: Apply the algorithm using the optimal number of clusters.
  4. Analyze Clusters: Examine the characteristics of each cluster to determine its defining features.

Let's start with Data Preparation. We'll consider features like PriceEuro, Range, and Efficiency for clustering.

Next, we'll determine the optimal number of clusters using the Elbow Method and Silhouette Score.

  1. The Elbow Method involves running the K-means clustering on the dataset for a range of values of ( k ) and then plotting the Sum of Squared Errors (SSE) for each value of ( k ). The "elbow" of the curve represents an optimal value for ( k ) (a balance between precision and computational cost).

  2. The Silhouette Score measures how close each point in one cluster is to the points in the neighboring clusters. Its values range from -1 to 1, where a high value indicates that the object is well matched to its own cluster and poorly matched to neighboring clusters.

Let's proceed with these methods to determine the optimal ( k ).

from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

# Elbow Method
sse = []
for k in range(1, 11):  # We'll check for up to 10 clusters
    kmeans = KMeans(n_clusters=k, random_state=42, n_init="auto")
    kmeans.fit(scaled_features)
    sse.append(kmeans.inertia_)

# Silhouette Score
sil_scores = []
for k in range(2, 11):  # Silhouette score is not defined for k=1
    kmeans = KMeans(n_clusters=k, random_state=42, n_init='auto')
    kmeans.fit(scaled_features)
    cluster_labels = kmeans.labels_
    sil_scores.append(silhouette_score(scaled_features, cluster_labels))

# Plotting the results
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(16, 6))

# Elbow Method
ax1.plot(range(1, 11), sse, marker='o')
ax1.set_title('Elbow Method')
ax1.set_xlabel('Number of Clusters')
ax1.set_ylabel('SSE')

# Silhouette Score
ax2.plot(range(2, 11), sil_scores, marker='o')
ax2.set_title('Silhouette Score')
ax2.set_xlabel('Number of Clusters')
ax2.set_ylabel('Silhouette Score')

ax1.locator_params(axis = "both", nbins = 15)
ax2.locator_params(axis = 'both', nbins = 15)

The dendrogram provides a visual representation of the hierarchical clustering. Each line represents a merger of two clusters, and the height of the line represents the distance (or dissimilarity) at which the merger occurred.

The dendrogram can be interpreted as follows:

  • Starting from the bottom, each leaf (or horizontal line) represents an individual data point (vehicle in our case).
  • As we move up, similar data points (or clusters) merge, represented by the vertical lines.
  • The height of the vertical lines indicates the dissimilarity at which the clusters merged. Higher vertical lines indicate greater dissimilarity.

By cutting the dendrogram horizontally at different heights, you can obtain different numbers of clusters. For instance, cutting it at the highest vertical line would give you two clusters, while cutting it lower could yield more clusters.
